Create a mobile-first web app called “Leah’s Legal Spider Solitaire”.

Core concept
This is a Spider Solitaire game combined with legal learning. The player solves card layouts while collecting evidence and answering legal questions. The experience must feel smooth, fast, and intuitive.

Main screen
Title at top: Leah’s Legal Solitaire
Buttons:
	•	Play
	•	Career Mode
	•	Casebook
	•	Flashcards
	•	Challenge a Friend

Design
	•	Clean UI
	•	Courtroom inspired theme
	•	Dark green, gold, and neutral tones
	•	Elegant serif fonts for headings
	•	Smooth animations
	•	Fast load time
	•	No clutter

Gameplay (Spider Solitaire)
	•	Use Spider Solitaire, not classic solitaire
	•	Easy mode default
	•	Highlight valid moves
	•	Auto-stack when obvious
	•	Smooth drag and snap cards
	•	Undo allowed in casual mode
	•	No lag or freezing

Game integration
	•	Each card move reveals “evidence fragments”
	•	Completing a full suit unlocks a legal question
	•	Clearing the board triggers a case decision
	•	Wrong answers reduce “credibility score”
	•	Perfect games give bonus rewards

Game modes
	1.	Career Mode

	•	Player starts as a law student
	•	Progress through cases
	•	Each level is a new client
	•	Unlock harder cases over time
	•	End goal is becoming a judge

	2.	Cross-Examination Mode

	•	Rapid-fire questions
	•	One mistake ends round
	•	High reward

	3.	Jury Decision Mode

	•	After a case, player decides guilty or not guilty
	•	Based on learned info

	4.	Time Pressure Court

	•	60 to 120 second rounds
	•	Solve cards + answer questions
	•	Leaderboard enabled

	5.	Focus Mode

	•	No timers
	•	No penalties
	•	Relaxed play

Learning system
	•	Questions based on:
	•	Criminal law
	•	Contract law
	•	Tort law
	•	AI generates new cases dynamically
	•	Cases must feel realistic with names and scenarios
	•	Difficulty adapts to player performance

Features:
	•	“Explain like I’m 5” button for simple explanations
	•	Mistake review system
	•	Show only incorrect questions
	•	Flashcard mode for revision
	•	Topic mastery tracking

Case system
	•	Every completed game unlocks a case
	•	Cases saved in “Casebook”

Casebook:
	•	Styled like a premium legal folder
	•	Each case shows:
	•	Title
	•	Outcome
	•	Key legal principle
	•	Difficulty
	•	Cases have rarity:
	•	Common
	•	Rare
	•	Legendary
	•	Hidden cases unlock via streaks or perfect games
	•	Cases must update regularly
	•	AI generates new, current-style cases

Progression system
	•	XP and leveling
	•	Reputation score

Levels:
	•	Law Student
	•	Junior Associate
	•	Senior Counsel
	•	Barrister
	•	Judge

Unlocks:
	•	New case types
	•	New visuals
	•	New card designs

Rewards
	•	Small rewards every few minutes
	•	Evidence bonuses
	•	XP boosts
	•	Streak rewards

Daily system
	•	Daily case challenge
	•	Same case for all players
	•	Rewards streak consistency

Events
	•	Weekly “Big Case”
	•	Limited-time themes
	•	Mystery cases unlocked by achievements

Random events during gameplay
	•	“New evidence discovered”
	•	“Witness unavailable”
	•	“Court objection”

These slightly change gameplay rules

Social features
	•	Challenge a friend via link
	•	Both play same case
	•	Compare score and verdict
	•	Debate mode:
	•	Two players argue sides
	•	Others vote
	•	Share results:
	•	Generate simple image with score and title

Customization
	•	Custom avatars like iMessage games:
	•	Skin tones
	•	Hairstyles including locs
	•	Outfits (suits, robes)
	•	Card themes:
	•	Classic courtroom
	•	Gold legal set
	•	Minimal

Monetisation
	•	No pay-to-win
	•	Cosmetic unlocks only
	•	Optional ads for rewards
	•	Expansion packs for new cases

Performance
	•	Must not lag
	•	Smooth animations
	•	Fast transitions
	•	Save progress instantly
	•	Resume mid-game

Accessibility
	•	Adjustable text size
	•	Colorblind-friendly cards

Offline support
	•	Core gameplay works offline

End-of-game screen

Win:
	•	“Case Closed”
	•	Show score, XP, and key legal fact

Lose:
	•	“Insufficient Evidence”
	•	Show correct answers with explanation

Appeal system
	•	If player loses
	•	One retry with harder questions
	•	Winning restores score

Final requirement

The app must feel simple to play, but deep over time.
Learning must feel natural, not forced.
Everything should be responsive, clean, and enjoyable.
